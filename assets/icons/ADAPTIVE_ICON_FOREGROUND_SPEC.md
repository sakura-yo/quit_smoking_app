# adaptive_icon_foreground.png 作成仕様

このファイルを差し替えるときに、再現性を保つための指示・仕様です。

---

## 1. 画像の基本仕様

| 項目 | 指定 |
|------|------|
| **ファイル名** | `adaptive_icon_foreground.png` |
| **保存場所** | `assets/icons/` |
| **画像サイズ** | **512 × 512 px**（必須） |
| **形式** | PNG、**アルファチャンネルあり（透過対応）** |

---

## 2. 色とレイヤーの関係

- **背景色は画像に含めない。**  
  アプリ側で `#2E7D32` の単色背景が敷かれるため、前景画像では「背景部分をすべて透過」する。
- **表示したいデザイン部分だけを不透明で描く。**  
  それ以外のピクセルは **完全透明（アルファ 0）** にすること。
- **デザインの色**  
  - 白（#FFFFFF）で描くのが無難。  
  - 他の色でもよいが、Android のマスクで円・角丸などに切り抜かれる前提で、背景 #2E7D32 とのコントラストを考慮する。

---

## 3. 表示のされ方（再現性のため）

- この画像は **背景 #2E7D32 の上にそのまま重ねて表示**される（inset 0%）。
- **端まで描いた部分はそのまま端まで表示**される。  
  余白をつけたい場合は、画像内で透明にして調整する。
- 左端に意図しない線やノイズが残ると目立つため、**左端は透明で揃える**とよい。

---

## 4. 作り直し時のチェックリスト

- [ ] キャンバスは **512 × 512 px**
- [ ] 背景は描かず **透過**
- [ ] デザイン部分だけ不透明（推奨：白 #FFFFFF）
- [ ] 保存形式は **PNG（透過あり）**
- [ ] ファイル名は `adaptive_icon_foreground.png`
- [ ] `assets/icons/` に上書き保存したあと、以下でランチャーアイコンを再生成する:
  ```bash
  dart run flutter_launcher_icons
  ```

---

## 5. デザイン内容を変えない場合（白だけの前景にしたい場合）

既存の `app_icon_512.png` から「白い部分だけ残して前景用に変換」する場合は、プロジェクトのスクリプトを使う:

```bash
dart run bin/icon_foreground_transparent.dart
```

- 入力: `assets/icons/app_icon_512.png`
- 出力: `assets/icons/adaptive_icon_foreground.png`
- 挙動: 緑系を透過し、デザイン部分を白（#FFFFFF）で前景画像として出力する。
